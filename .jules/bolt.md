## 2024-04-20 - Removed String Allocations From Scanner Hot Path
**Learning:** In the inner loop of `ScanEngine::process_file_stream`, literal keyword searches were allocating string clones and performing `to_lowercase` on both the input string AND every single search keyword for every single line. This O(N * M) allocation overhead on the hot path caused substantial and unnecessary memory pressure.
**Action:** Always verify if keyword pre-computation or case conversions can be hoisted out of a tight IO loop, especially in a fast language like Rust, and avoid mapping or cloning inside hot read loops unless strictly required by mutation semantics.
## 2024-04-20 - Reduced React Re-Renders By Throttling Zustand SSE Streams and Avoiding ES6 Map
**Learning:** High-frequency Server-Sent Events (SSE) updating Zustand state with ES6 Maps causes severe rendering performance degradation. React components cannot efficiently evaluate partial state updates of Maps, and doing so on every streaming data chunk triggers too many reconciliation cycles.
**Action:** For high-frequency state updates in React via Zustand, always prefer plain objects (Records) over Maps, and strictly implement throttle buffers (e.g. 100ms batched state writes via `setTimeout`) for streaming operations like SSE or WebSocket events. Ensure the buffer is fully flushed on stream closure to avoid data loss.
