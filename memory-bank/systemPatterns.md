# System Patterns

## Architecture
The project follows a client-server architecture:
- **Frontend**: A web application built with React/TypeScript.
- **Backend**: A service written in Rust.

## Key Technical Decisions
- Use of Rust for the backend to ensure performance and safety.
- Use of TypeScript/React for the frontend for type safety and component-based architecture.
- Separation of concerns between frontend and backend.

## Design Patterns
- Modular architecture with clear separation between frontend and backend components.
- The use of a centralized `memory-bank/` for documentation.
