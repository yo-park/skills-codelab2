Design a single-executable tool that allows users to upload Java (Logback/Log4j) log files via a local web UI, enabling high-performance analysis through memory streaming without disk storage for large-scale files.
The application should feature a Rust backend and a React frontend with the following layout:

|                    | - multi-file input                       |
|                    | - concurrency                            |
|                    | - max matches per file                   |
|                    | - case sensitivity                       |
|                    | - pattern mode                           |
|                    |------------------------------------------|
|                    | Keywords Area                            |
|                    | - textarea                               |
|                    | - context lines                          |
|                    |------------------------------------------|
|                    | Progress Area                            |
|                    | - per-file progress bar                  |
|                    | - bytes / lines / matches                |
|                    |------------------------------------------|

|                    | - filter bar                             |
|                    | - result table                           |
|                    | - csv download                           |
